Which files to download

Scanned texts have two PDFs: the plain image scan and a larger searchable version with an OCR text layer. “PDF (image scan)” grabs just the image scan (no text layer); “Searchable text PDF” grabs only the OCR’d one.

Help & features

IA Desktop is a friendly front-end for the Internet Archive — search, browse, download, and upload without the command line. Here's a tour.

Searching

Type anything in the search box and press Enter. A few ways to narrow things down:

  • Field keywords — prefix a word with a field name and a colon to search just that field. Combine as many as you like:
    • title: — words in the title, e.g. title:housewife
    • subject: — topic/subject tag, e.g. subject:"world war"
    • creator: — author or uploader, e.g. creator:tolkien
    • language: — e.g. language:japanese
    • mediatype: — texts, audio, movies, image, software, data, web
    • date:, collection:, identifier:
    Use quotes for multi-word values: subject:"silent film". Words with no keyword are searched everywhere.
  • Year range — the from YYYY / to YYYY boxes next to the search field limit results to that publication-date range. You can also narrow to a month by typing YYYY-MM (or YYYY-M) — e.g. 1940-09 for September 1940. A bare year covers the whole year; a month covers that whole month (from is rounded down to the 1st, to up to the last day).
  • Sort — the dropdown re-orders by best match, downloads, or date.
  • Advanced — the Advanced ▾ button opens a form with separate title/subject/creator/language/date fields and media-type checkboxes, and shows the exact query it builds.

Filtering & browsing results

  • Facets — the left sidebar lets you click a media type, year, subject, language, or collection to filter the current results. Subjects show the top 30 with a More… button.
  • Filter box — the “Filter these results by title…” box narrows the items already on screen as you type (no new search).
  • Grid / List — toggle between preview cards and a compact list. In Preferences you can also set the density (Comfortable / Cozy / Compact).
  • Subject tags — turn on “Subject tags” to show topic chips under each title.
  • Per page & paging — show 50, 100, or 200 results per page, step with Prev/Next, or type a page number and hit Go.

Selecting & downloading

  • Tick the checkbox on any result to select it. Shift-click a second checkbox to select the whole range between them.
  • Select all on page ticks everything visible; Deselect all clears your entire selection. Selections are kept as you page, so you can gather items across several pages before downloading.
  • Download selected downloads every selected item. A single item's Download button grabs just that one.
  • Which files you get, the filename format, and the download folder are all set in Preferences. Files download one at a time and failed transfers retry automatically.
  • Already-downloaded files are skipped — if a file with the same name is already in the download folder, it's assumed downloaded and skipped, so re-running a download (or a whole collection) only fetches what's missing. To force a fresh copy, turn on “Re-download files that already exist” in Preferences.

Working with results

  • Click a creator under a title to search for more by that creator; click a subject tag to search that subject — both run a new search inside the app.
  • Right-click a result for Copy Title and Copy Creator.
  • Hover a title to see the full title as a tooltip when it's truncated.
  • In an item's Details, More by… and the Collection: buttons run those searches in the app rather than opening the website.

Favorites, uploads & more

  • ★ Favorite — the star on a card (or in its detail view) saves an item to the Favorites tab.
  • Details — opens an item's full metadata, file list, and (for collections) its member items. Edit metadata and Tasks appear only on items you uploaded.
  • Logs — Preferences → turn on logging & diagnostics (off by default), then Open logs folder reveals a timestamped log of downloads, uploads, and errors.

Uploading from a spreadsheet (CSV)

To create many items at once, prepare a CSV and use Upload → Bulk upload from a spreadsheet. Each row becomes one new archive.org item.

  • Required columns: identifier (the unique item id — lowercase, no spaces) and file (the file to upload, relative to the CSV's own folder). To attach several files to one item, repeat the file column (file, file2, file3…) or list the files on extra rows that share the same identifier.
  • Metadata columns: add a column per metadata field — title, creator, date, subject, description, language, collection, mediatype, etc. Each column header is the field name; the cell is its value. Separate multiple subjects with semicolons.
  • Paths: file paths are resolved relative to the folder the CSV is in, so keep the spreadsheet next to your files (or use absolute paths).
  • Pick the CSV in the Upload tab to see a parsed preview (with any missing files flagged) before anything is uploaded. Items upload one at a time.
